## 工作流程

1. **信息收集**：通过豆瓣等网站获取书籍信息
2. **内容撰写**：撰写约5000字的书评，包含书籍信息、核心内容、核心洞见、推荐建议等
3. **PDF生成**：使用pandoc + weasyprint生成PDF
4. **文件保存**：保存至本地文件目录和GitHub仓库
5. **Git提交**：每篇书评独立commit推送

## 技术细节

- **文件格式**：Markdown + HTML + PDF
- **生成工具**：pandoc + weasyprint
- **版本控制**：Git + GitHub
- **文件命名**：书评_书名.md / 书评_书名.pdf

## 书评特点

1. **结构化**：统一的章节结构（书籍信息、核心内容、核心洞见、推荐建议等）
2. **信息丰富**：包含作者、出版社、出版时间、豆瓣评分等信息
3. **评价客观**：根据书籍内容、学术价值、可读性等因素综合评分
4. **推荐具体**：针对不同读者群体提供阅读建议
